# Piclaw — features matrix (as-built audit)

| Wanted capability | Implemented? | Where in code (file + function) | Notes / constraints / safety gates |
|-------------------|---------------|----------------------------------|------------------------------------|
| Identity (device_id) + persistent soul (/opt/piclaw_identity) | Yes | `core/identity.js` loadIdentity() (device_id, .device_identity.json in runtime); `identity_bridge/paths.js` getRoot(), selfPath(), goalsPath(), etc. (identity root); `identity_bridge/readers.js` loadSelf(), loadGoals(); `identity_bridge/writers.js` atomicWrite, appendExperience, writeSuggestions, etc. | device_id from core/identity (runtime file); soul (mission, goals, experiences) in /opt/piclaw_identity. Override via PICLAW_IDENTITY_PATH. |
| Goal review loop + suggestions | Yes | `goal_loop/scheduler.js` start(); `goal_loop/review.js` runReview(); `goal_loop/suggestions.js` generateSuggestions(); `goal_loop/evaluator.js` evaluateGoals(); `goal_loop/health_gate.js` runHealthGate(). | Interval: PICLAW_GOAL_REVIEW_INTERVAL_HOURS (default 6). First run 10 min after boot. Health gate: low disk or runtime not writable skips review. Suggestions: update, integration setup, diagnostic (e.g. /probe_uart). |
| Update discovery + A/B update invoke | Yes (discovery + invoke); partial (rollback) | `update_probe/scheduler.js` startUpdateScheduler(notifyFn); `update_probe/check_remote.js` checkRemote() (GitHub or PICLAW_UPDATE_URL); `core/update_channel.js` requestUpdate() → execSync("piclaw-update"); `scripts/piclaw-update.sh` (slot switch, health wait, revert symlink on failure). | Discovery: PICLAW_UPDATE_SOURCE=github|url. Notify once per new version to PICLAW_TELEGRAM_CHAT_ID. /update runs piclaw-update. Rollback: script reverts symlink if .boot-ok not seen in 20s; no persistent rollback guard or automatic retry limit. |
| Telegram /status | Yes | `comms/telegram.js` bot.onText(/\/status/); piclaw.js buildStatusText(). | Identity, env, system, process, version, integrations, economy, hardware (if Pi). |
| Telegram /whoami | Yes | `comms/telegram.js` bot.onText(/\/whoami/); piclaw.js buildWhoamiText(). | device_id, mission, goals summary from identity_bridge. |
| Telegram /review_status | Yes | `comms/telegram.js` bot.onText(/\/review_status/); piclaw.js buildReviewStatusText(); identity_bridge getLastReview(). | Last review at, result, duration_ms, reason. |
| Telegram /update | Yes | `comms/telegram.js` bot.onText(/\/update/); options.requestUpdate() → core/update_channel.requestUpdate(). | Runs piclaw-update; friendly message if not found (A/B not set up). |
| Telegram /gpio | Yes | `comms/telegram.js` bot.onText(/\/gpio.../); options.gpioControl getControlConfig, pulsePin, setPinFor; `hardware/gpio_control.js`. | Help, /gpio pulse &lt;pin&gt; &lt;ms&gt;, /gpio set &lt;pin&gt; HIGH\|LOW &lt;sec&gt;. Guarded: PICLAW_GPIO_CONTROL_ENABLED=1, whitelist, max ms/sec, cooldown; Pi only (detect_platform). |
| Telegram /probe_uart | Yes | `comms/telegram.js` bot.onText(/\/probe_uart/); options.runUARTProbe (uartWatch.pause → uartProbeBridge.runUARTProbe → resume); options.identifyUartDevice (uart_identity/matcher). | Pauses UART watch, runs Python probe, identifies device (matcher), resumes watch. |
| Telegram /uart_devices | Yes | `comms/telegram.js` bot.onText(/\/uart_devices/); piclaw.js buildUartDevicesText(); uart_identity/registry.load(). | List devices from uart_registry.json (identity). |
| Telegram /uart_label | Yes | `comms/telegram.js` bot.onText(/\/uart_label\s+(\S+)\s+(.*)/); piclaw.js setUartLabel(id, label); uart_identity/registry load/save, identityBridge.appendExperience. | Set human-readable label on device; id from /uart_devices. |
| Telegram /hw | Yes | `comms/telegram.js` bot.onText(/\/hw/); options.getHardwareState() → hardware/hardware_state.getHardwareState(). | UART/GPIO summary and last events; Pi only (else "n/a (not Raspberry Pi)"). |
| Telegram /twitter | Yes | `comms/telegram.js` bot.onText(/\/twitter/); options.getTwitterStatus() → integrations/twitter_api_bridge.getTwitterStatus(). | Read-only check; uses extension. |
| Telegram /github | Yes | `comms/telegram.js` bot.onText(/\/github/); options.getGitHubStatus() → integrations/github_api.getGitHubAuthStatus(). | PAT auth check; login, rate limit. |
| Telegram /mailtest | Yes | `comms/telegram.js` bot.onText(/\/mailtest/); options.sendTestMail() → integrations/smtp_api.sendTestMail(). | Sends to PICLAW_SMTP_TEST_TO only. |
| UART monitoring + probing + device registry + labeling | Yes | `hardware/uart_watch.js` startUARTWatch (createReadStream /dev/serial0 or /dev/ttyAMA0), pauseUARTWatch, resumeUARTWatch; `integrations/uart_probe_bridge.js` runUARTProbe(); `uart_identity/matcher.js` identifyDevice(); `uart_identity/registry.js` load/save (identity uart_registry.json); setUartLabel in piclaw.js. | Pi-only paths; watch runs only when isRaspberryPi(). Registry in identity; MAX_UART_DEVICES 64; fingerprint + signature_hash. |
| GPIO monitoring + event rules + guarded GPIO output | Yes | `hardware/gpio_watch.js` startGPIOWatch (gpiomon), PICLAW_GPIO_PINS; `events/engine.js` handleGPIOEvent; `events/rules.js` getRules() from rules.json; `hardware/gpio_control.js` pulsePin, setPinFor (gpioset), whitelist, cooldown. | Rules: gpio[pin].on (rising/falling), notify, cooldown_sec. Output: PICLAW_GPIO_CONTROL_ENABLED, PICLAW_GPIO_OUTPUT_WHITELIST, max ms/sec, cooldown. Pi only. |
| Gateway auth (Telegram initData) + endpoints /status, /devices, /review | Yes | `gateway/auth.js` validateInitData(), isAllowed(); `gateway/server.js` authMiddleware, GET /status (gateway_api.getStatusJson), GET /devices (gateway_api.getDevices), GET /review (gateway_api.getReview). | Auth: HMAC-SHA256(WebAppData, token); optional owner check (PICLAW_MINI_APP_OWNER_TELEGRAM_ID or self.owner). CORS: Access-Control-Allow-Origin *; bind 0.0.0.0. |
| Integrations: GitHub auth check | Yes | `integrations/github_api.js` getGitHubAuthStatus(); `integrations/github_identity.js` isConfigured(). | PICLAW_GITHUB_PAT, PICLAW_GITHUB_USERNAME. |
| Integrations: SMTP test | Yes | `integrations/smtp_api.js` sendTestMail(); `integrations/mail_identity.js` isConfigured(). | PICLAW_SMTP_*; recipient PICLAW_SMTP_TEST_TO only. |
| Integrations: Twitter read-only check | Yes | `integrations/twitter_api_bridge.js` getTwitterStatus(); `integrations/twitter_identity.js`; extension extensions/twitter_api. | PICLAW_TWITTER_*; Python extension. |
| Integrations: Moltbook placeholders | Partial | `integrations/moltbook_identity.js` isConfigured() (PICLAW_MOLTBOOK_TOKEN). | Presence only; no API calls in repo. |
| Self-protection: path safety, no writes outside runtime/identity | Partial | `core/self_guard.js` SAFE_ROOT, isPathSafe(); `introspection/filesystem_view.js` listRuntimeDir uses isPathSafe; `goal_loop/health_gate.js` uses SAFE_ROOT for write test. Identity writes only via identity_bridge/paths.js (all under PICLAW_IDENTITY_PATH). Runtime writes: .device_identity.json, heartbeat.json, .boot-ok, update_state.json, .goal_loop_write_test (temporary). | No single central guard that blocks all fs writes. Identity_bridge writers use paths.js only (identity dir). Runtime can write under its own dir; no explicit guard against writing elsewhere outside those modules. |
| Power loss recovery signals (heartbeat gap logging) | Yes | `core/watchdog.js` writes heartbeat.json every 30s; `piclaw.js` main() reads heartbeat, if last_seen older than STALE_HEARTBEAT_MS (90_000) logs "recovered from unexpected shutdown". | No persistent log of gaps; single log line on next boot. |
| "Always aware" sensors: wifi, connectivity, throttling; Pi-only | Yes (sensors); Pi-only where noted | `sensors/wifi_status.js` getWifiStatus() (iw dev wlan0 link); `sensors/connectivity.js` checkConnectivity() (DNS api.openai.com); `sensors/power_status.js` getPowerStatus() (vcgencmd get_throttled). piclaw.js senseEnv() every 30s, runtime_state.environment. | wifi_status/connectivity: run on any platform (iw/vcgencmd fail gracefully on non-Linux/non-Pi). power_status: returns { unknown: true } on non-Pi. Throttling/undervoltage only meaningful on Pi. |
