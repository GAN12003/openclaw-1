NetHunter hlte: minimal chroot — run in this order (inside bootkali)

0a) chroot-continue.sh
   (All-in-one non-destructive: snapshot, audit, both minimize dry-runs, export of top 50 largest packages to snapshots/last-top-packages.txt. CHROOT_CONTINUE_SKIP_SNAPSHOT=1 to skip 0b.)

0b) chroot-snapshot.sh
   (If you did not use 0a. Keep the printed folder path.)

0c) chroot-export-top-packages.sh 80
   (Optional. Edit the N argument; output under snapshots/last-top-packages.txt by default. Use names as hints, not as blind purge.)

Restore: rollback is manual and risky. A snapshot is for reference; do not dpkg --set-selections on a half-broken chroot without reading apt/dpkg error output first. Prefer: repair with chroot-cleanup-finish.sh, then remove fewer packages.

When dpkg is stuck in a loop (your log):
- systemd: "Failed to enable units" / "Protocol driver not attached" — expected in a chroot (no PID1). Run chroot-dpkg-repair.sh first (policy-rc.d, SYSTEMD_OFFLINE, machine-id).
- exim4-config: adduser "Unknown option: allow-bad-names" — tooling mismatch; chroot-dpkg-repair tries adduser/passwd refresh; if it still fails, purge exim4-* and use msmtp-mta or no local MTA (read chroot-dpkg-repair.sh footer).
- Last resort (only if still stuck): ln -sf /bin/true /usr/bin/systemctl && dpkg --configure -a  then REMOVE that symlink if a real systemctl exists — usually not needed on NetHunter.

0z) chroot-dpkg-repair.sh
   (Unstick systemd+deps and exim round; then chroot-cleanup-finish.sh)

1) chroot-audit.sh
   (Note disk free, dpkg --audit, largest packages.)

2) chroot-minimize.sh --profile safe
   (Dry-run. Then: chroot-minimize.sh --profile safe --apply)

3) chroot-minimize.sh --profile minimal
   (Dry-run. Edit scripts/nethunter-hlte/packages-minimal.list first: comment anything you use.)
   (Then: chroot-minimize.sh --profile minimal --apply)

4) chroot-purge-docs-dry.sh
   (Dry-run. Optional: chroot-purge-docs-dry.sh --apply  — reclaims a lot of space; dev docs go away.)

5) chroot-cleanup-finish.sh
   (Repairs dpkg, fixes dependencies, autoremove, clean cache. Safe to re-run.)

6) chroot-audit.sh again
   (Confirm package count and free space moved in the right direction.)

Never remove blindly: if apt wants to remove nmap/git/curl/wget, stop and fix the list.
If a dry-run tries to install courier-mta, replace lightdm, or remove kali-undercover, hit Ctrl+C and do not apply — exim/MTA must be untangled manually.
chroot-minimize no longer simulates autoremove unless you add --autoremove (huge surges of purges on broken dpkg are common).
